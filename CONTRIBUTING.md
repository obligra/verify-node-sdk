# Contributing

Thank you for your interest in the Obligra Verify Node.js SDK.

This repository is currently in early integration preview. External contributions may be limited while the SDK contract, API surface, and release process are finalized.

## Development Principles

- Keep the SDK simple and predictable.
- Do not expose internal Obligra platform mechanics.
- Do not commit secrets, API keys, internal URLs, or customer data.
- Prefer clear examples over clever abstractions.
- Treat retained decision records as operational evidence, not generic logs.

## Pull Requests

Before opening a pull request:

1. Open an issue describing the proposed change.
2. Keep the change focused.
3. Include tests when changing SDK behavior.
4. Update documentation when changing public methods, configuration, or examples.

## Security

Do not report security issues through public GitHub issues. See `SECURITY.md`.
